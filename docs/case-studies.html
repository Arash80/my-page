Reliable AI

MetricTrace

An auditable metric-computation engine that keeps deterministic calculations separate from optional natural-language explanations.

Raw inputs→Pydantic validation→Deterministic metrics→CLI · Streamlit · JSON

Problem

When an LLM performs both calculation and explanation, results are harder to reproduce, validate, and audit.

My contribution

Designed the typed schemas, calculation boundary, structured evidence model, CLI and Streamlit delivery paths, and optional grounded explanation layer.

Result

A single deterministic computation layer supports both CLI and Streamlit through one validated JSON contract, eliminating duplicated metric logic and keeping results consistent across interfaces.

Python · Pydantic · LangChain · Streamlit · CLI · pytest
MLOps

Fraud ML Platform

A production-oriented fraud modeling workflow designed around time-aware evaluation, uncertainty, and operational monitoring.

Temporal data→Airflow · dbt→Models · Calibration→FastAPI · Drift

Problem

Random validation and accuracy-only reporting can hide temporal leakage, minority-class failure, and changes in production data.

My contribution

Connected temporal validation, LightGBM and XGBoost experiments, calibration, conformal prediction, MLflow tracking, orchestration, API serving, and deployment components.

Result

Three operational layers—data, modeling, and serving—are represented in one reproducible platform workflow.

LightGBM · XGBoost · Airflow · MLflow · FastAPI · PostgreSQL · Docker
Agentic AI

Advanced Agentic RAG

A document-intelligence testbed for comparing retrieval strategies and routing weak-evidence questions through corrective steps.

User query→Retrieve · Rerank→Evidence gate→Answer · Citation · Trace

Problem

Retrieval quality varies by query, document structure, and evidence density; one search strategy is not consistently sufficient.

My contribution

Implemented dense, sparse, hybrid, MMR, multi-query, and parent-document retrieval with reranking, evidence checks, corrective routing, citations, guardrails, and traces.

Result

Six retrieval strategies can be examined through one evaluation and observability workflow.

LangChain · Qdrant · BM25 · Reranking · Phoenix · Guardrails
Local RAG

DeskPilot

Problem

IT support knowledge is often fragmented across structured records and internal documents that should remain local.

My contribution

Result
