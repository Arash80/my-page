Engineering projects

Reliable AI

MetricTrace

An auditable metric-computation engine that keeps deterministic calculations separate from optional natural-language explanations.

Raw inputs→Pydantic validation→Deterministic metrics→CLI · Streamlit · JSON

Problem

When an LLM performs both calculation and explanation, results are harder to reproduce, validate, and audit.

My contribution

Designed typed schemas, the deterministic calculation boundary, structured evidence, CLI and Streamlit delivery paths, and an optional grounded explanation layer.

Engineering result

One validated computation engine supports multiple interfaces through a stable JSON contract, avoiding duplicated metric logic.

Python · Pydantic · LangChain · Streamlit · CLI · pytest
MLOps

Fraud ML Platform

A production-oriented fraud modeling workflow designed around time-aware evaluation, uncertainty, and operational monitoring.

Temporal data→Airflow · dbt→Models · Calibration→FastAPI · Drift

Problem

Random validation and accuracy-only reporting can hide temporal leakage, minority-class failure, and changes in production data.

My contribution

Connected temporal validation, LightGBM and XGBoost experiments, calibration, conformal prediction, MLflow tracking, orchestration, API serving, and deployment components.

Engineering result

Data preparation, modeling, evaluation, monitoring, and serving are represented in one reproducible platform workflow.

LightGBM · XGBoost · Airflow · MLflow · FastAPI · PostgreSQL · Docker
Agentic AI

Advanced Agentic RAG

A document-intelligence testbed for comparing retrieval strategies and routing weak-evidence questions through corrective steps.

User query→Retrieve · Rerank→Evidence gate→Answer · Citation · Trace

Problem

Retrieval quality varies by query, document structure, and evidence density; one search strategy is not consistently sufficient.

My contribution

Implemented dense, sparse, hybrid, MMR, multi-query, and parent-document retrieval with reranking, evidence checks, corrective routing, citations, guardrails, and traces.

Engineering result

Retrieval strategies can be compared through one evaluation and observability workflow with explicit weak-evidence handling.

LangChain · Qdrant · BM25 · Reranking · Phoenix · Guardrails

AI Agents

Reflexion Agent

A bounded LangGraph workflow that drafts, critiques, researches, and revises answers while preserving structured state.

Problem

Unbounded self-reflection can increase cost and complexity without guaranteeing better evidence or a clear stopping condition.

My contribution

Designed explicit graph nodes, iteration limits, Pydantic state, search and revision tools, citation URL checks, retries, and node-level tests.

Engineering result

The agent follows a controlled improvement loop that is observable, testable, and easier to debug than an open-ended autonomous process.

LangGraph · Pydantic · Tool Calling · Validation · pytest

Cyber-Physical Systems

Attack and Fault Detection

Observer- and residual-based methods for detecting and isolating abnormal behavior in nonlinear cyber-physical systems.

System problem

Research contribution

The work combines state estimation, robust control concepts, security modeling, and quantitative detection metrics for safety-critical dynamic systems.
